# 开发规范

## 文件命名

项目内新增文件默认使用小写横杠命名，也就是 kebab-case。

推荐：

```txt
mahjong-room-design.md
wechat-login.js
board-graphic.js
main-controller.js
```

不推荐：

```txt
MahjongRoomDesign.md
wechatLogin.js
board_graphic.js
MainController.js
```

## 适用范围

- 文档文件：`*.md`
- 前端源码：`*.js`、`*.ts`
- 服务端源码：`*.ts`
- 配置辅助文件和脚本文件

## 例外

以下情况可以沿用生态或框架约定：

- 第三方工具要求的固定文件名，例如 `README.md`
